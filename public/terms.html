6. Message & File Ephemerality

Messages and files on BitMask are designed to be ephemeral:

  • Messages and files are held in volatile server memory only during transit
  • All content self-destructs on the recipient's device after the sender-configured TTL
  • No messages or files are written to any persistent storage (disk, database, or backup) on the server
  • File transfers are limited to 5 MB per file

However, we cannot prevent a recipient from capturing content before it self-destructs (e.g., via screenshots, screen recording, downloading files, or browser developer tools). Send sensitive information at your own risk.

7. Identity Persistence & Security

BitMask offers optional identity persistence features:

  • Remember Identity: Saves your identity to browser localStorage. You are responsible for the security of your own device.
  • Passphrase Lock: Encrypts your stored identity with a passphrase you choose. If you forget it, your identity is irrecoverable — we cannot reset it.
  • Export/Import: Allows you to download and restore your identity. Exported files may optionally be passphrase-encrypted. Protect exported files as you would a private key.

These features operate entirely client-side. We have no ability to recover, reset, or access your identity.

8. Group Rooms

BitMask provides encrypted group rooms. By creating or joining a group room, you acknowledge:

  • Room Codes: A 6-character room code is generated for each group. Anyone with the code can join (up to 20 members). Treat room codes like passwords — share them only with intended participants.
  • Group Key: The room host generates and distributes an AES-256 group encryption key. All members sharing this key can decrypt group messages. If a member leaves, the current group key remains in use unless a new room is created.
  • Host Role: The room creator is the host. If the host disconnects, host privileges transfer automatically. The host role only affects key distribution — the host cannot decrypt anything the group key doesn’t already allow.
  • Member Visibility: All members of a group room can see the Peer IDs of other members in the room.
  • No Key Rotation: When a member leaves, the group key is not automatically rotated. For maximum security, create a new room if a participant is compromised.
